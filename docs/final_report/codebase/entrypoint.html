entrypoint

Module containing the main entrypoint to the program start() as well as the Manager which schedules and executes the other parts of the whole project to fulfill their purposes

class p2p_nse5.entrypoint.Manager(conf: Configuration)

Manager of program sub-tasks, executor of the NSE server, NSE scheduler and Gossip client

Parameters

conf – full package configuration instance

async _schedule_nse_round()

Schedule the next NSE round and call continue recursively

This recursive NSE round scheduler calls itself in regular intervals, depending on the setting of the NSE frequency. It will trigger one p2p_nse5.nse.RoundHandler.run() call at the start of a new NSE round.

Returns

None

_send_gossip_announce(data: bytes) → bool

Write arbitrary data as gossip announcement to the current network transport to the gossip API

This function must be called in the context of an async function.

Parameters

data – arbitrary data in bytes

Returns

bool whether writing the data completed successfully

async _start_gossip_client(loop: AbstractEventLoop, first: bool = False) → bool

Start a new instance of the Gossip API client class

Note that this method adds itself to the event loop again if a later Gossip connection suddenly fails. It will wait exponentially longer the more re-connect attempts have previously failed.

Parameters

  • loop – running AsyncIO event loop which should execute the Gossip client

  • first – switch to determine whether the current start is the first start of the Gossip client in the program’s runtime

Returns

whether a new Gossip client has successfully established a connection

Raises

Exception – when the very first connection to the Gossip API fails

reconnect_client()

Create an instant task in the event loop to (re)connect the Gossip client

Returns

None

async run()

Main program routine

This method contains a blocking call to asyncio.events.AbstractServer.serve_forever() to keep the current event loop running. It’s also the executor of the NSE API server, listening for incoming connections which uses p2p_nse5.nse.Protocol as the protocol handler. It will start the recursive NSE scheduler _schedule_nse_round() once at startup, after successfully having established a Gossip connection via _start_gossip_client().

Returns

does not return while the Manager executes, but will be quit via KeyboardInterrupt
